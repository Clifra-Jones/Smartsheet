Add-SmartsheetColumn

Add a column to a Smartsheet

Adds a new column to a smartsheet. Column can be specified as a column object or properties parameters.
If is column index already exists the column will be inserted at that position. Columns after that index will have their index
incremented by 1.

Syntax 

Add-SmartsheetColumn -Id <String> -ColumnId <String> -Index <Int32> [-title <String>] [-description <String>] [-type <String>] [-formula <PSObject>] [-hidden <Boolean>] [-autoNumberFormat <PSObject>] [-contactOptions <PSObject>] [-format <String>] [-locked <Boolean>] [-options <String[]>] [-symbol <String>] [-systemColumnType <String>] [-validation <Boolean>] [-version <Int32>] [-width <Int32>] [<CommonParameters>]

Add-SmartsheetColumn [-Id <String>] -column <PSObject> -ColumnId <String> -Index <Int32> [-title <String>] [-description <String>] [-type <String>] [-formula <PSObject>] [-hidden <Boolean>] [-autoNumberFormat <PSObject>] [-contactOptions <PSObject>] [-format <String>] [-locked <Boolean>] [-options <String[]>] [-symbol <String>] [-systemColumnType <String>] [-validation <Boolean>] [-version <Int32>] [-width <Int32>] [<CommonParameters>]

Parameters

Name Alias Type Description Required? Pipeline Input Default Value
-Id String Id of the Smartsheet containing the column. true false
-column PSObject A Smartsheet column object. Cannot be used with column property parameters. true true (ByValue)
-ColumnId String Id of teh column to update. true false
-Index Int32 Index if the column to update. true false 0
-title String Column Title false false
-description String Column description false false
-type String Column type false false
-formula PSObject The formula for a column, if set, for instance =data@row. false false
-hidden Boolean Indicates visibility of the column. false false False
-autoNumberFormat PSObject Object that describes how the the System Column type of "AUTO_NUMBER" is auto-generated. false false
-contactOptions PSObject Array of ContactOption objects to specify a pre-defined list of values for the column. Column type must be CONTACT_LIST.
The contact option object is in the form:
email = {email address}
name = {contact name}		 false false
-format String Format string. false false
-locked Boolean Indicates whether the column is locked. A value of true indicates that the column has been locked by the sheet owner or the admin. false false False
-options String[] Array of the options available for the column. false false
-symbol String When applicable for CHECKBOX or PICKLIST column types. false false
-systemColumnType String If this is a system column what type is it. false false
-validation Boolean Indicates whether validation has been enabled for the column (value = true). false false False
-version Int32 0: CONTACT_LIST, PICKLIST, or TEXT_NUMBER.
1: MULTI_CONTACT_LIST.
2: MULTI_PICKLIST.		 false false 0
-width Int32 Display width of the column in pixels. false false 0

Outputs

The output type is the type of the objects that the cmdlet emits.

  • An updated column object.

Examples

EXAMPLE 1 
To add a new colum to a Smartsheet.
PS> $newColumn = $Sheet | Add-SmartsheetColumn -title "Title" -type:TEXT_NUMBER -description 'My new column'
EXAMPLE 2 
To insert a new column at position 4 (columns after position 4 are shifted to the right and thier index incremented).
PS> $newColumn = $Sheet | Add-SmartsheetColumn -title "Asset" -type:TEXT_NUMBER -Description "Fixed asset" -index 4
EXAMPLE 3 
Add a new column with contact objects.
PS> $contacts = @(
    @{
        email = "johndoe@example.com"
        name = "John Doe"
    },
    @{
        email = "janedoe@example.com"
        name = 'Jane Doe
    }
)
PS> $newColumn = $Sheet | Add-SmartsheetColumn -title "EmployeeName" -type:TEXT_NUMBER -contactOption $contacts

Export-SmartSheet

Exports a powershell array into a new Smartsheet.

Exports an array of PSObjects into a smartsheet. This function will always create a new sheet even if
there is a sheet of the same name. The API will attempt to determine column types.
To prevent Sheets of the same name being created, use the -overwriteAction and -overwriteSheetId parameters.

Syntax 

Export-SmartSheet [-InputObject <PSObject>] -SheetName <String> [-Folder <String>] [-headerRow <Int32>] [-primaryColumn <Int32>] [<CommonParameters>]

Export-SmartSheet [-InputObject <PSObject>] -SheetName <String> [-Folder <String>] [-headerRow <Int32>] [-primaryColumn <Int32>] -overwriteAction <String> -overwriteSheetId <String> [<CommonParameters>]

Parameters

Name Alias Type Description Required? Pipeline Input Default Value
-InputObject PSObject Object to create the Smartsheet from false true (ByValue)
-SheetName String The name of the new Smartsheet. true false
-Folder String The name and path to the folder to create the Smartsheet in in the for folder1/folder2/folder3
folder(s) must exist.		 false false
-headerRow Int32 Row to use for column headers.
All rows above this row are ignored.
If ommitted the first row will be used. A value of -1 will crete defaule headers in the form Column1, Column1, etc.		 false false 0
-primaryColumn Int32 The column to use as the primary column. default is the 1st column. false false 0
-overwriteAction String What to do of the sheet name already exists. Choices are Replace or Rename.
Multiple sheets can have the same name in a folder. If you omit this parameter a sheet with the same name will be created.		 true false
-overwriteSheetId String Because tou can have multiple sheets with the same name (with different sheet Ids) you must provide the sheet Id for the overwrite action. true false

Examples

EXAMPLE 1

EXAMPLE 2

Get-SortedSmartsheet

Sort rows in a Smartsheet.

Sort the rows in a smartsheet.

Syntax 

Get-SortedSmartsheet -id <String> -columnId <String> [<CommonParameters>]

Get-SortedSmartsheet -id <String> [-sortCriteria <PSObject[]>] [<CommonParameters>]

Get-SortedSmartsheet -id <String> [-Ascending] [<CommonParameters>]

Get-SortedSmartsheet -id <String> -Descending [<CommonParameters>]

Parameters

Name Alias Type Description Required? Pipeline Input Default Value
-id String Id of the sheet to srot rows in. true true (ByPropertyName)
-sortCriteria PSObject[] An array of sort criteria objects. The objects shoud have 2 properties, columnId and direction (ASCENDING or DESCENDING) false false
-columnId String Id of the column to sort on for a single column sort. true false
-Ascending SwitchParameter Sort the column Ascending (default) false false False
-Descending SwitchParameter Sort the column Descending true false False

Outputs

The output type is the type of the objects that the cmdlet emits.

  • Sheet object with the results of the sort operation.

Examples

EXAMPLE 1 
How to create a multi-sort sortCriteria object.
I this example we are going to sort a Smartsheet of employee salary information by Department and Salary in descending order.
To create the sort Criteria, 1st create an array.
PS> $sortCriteria - @()
The create a criteria objects from the columns collection of a sheet object named $Sheet.
PS> $criteria - [PSCustomObject]@{
    columnId = $Sheet.columns.Where({$_.title -eq "Department"}).ColumnId
    direction = "ASCENDING"
}
Add this to the array.
$sortCriteria += $criteria
Create another criteria object
PS >$criteria = [PSCustomObject]@{
    columnId = $Sheet.Columns.Where({$_,title -eq "Salary"}).ColumnId
    direction = "DESCENDING"
}
Add this to the array
PS >$sortCriteria += $criteria
Now sort the sheet.
PS >$SortedSheet = $sheet | Get-SortedSmartSheet -SortCriteria $sortCriteria
